# 独居测试和群居测试

独居测试是指，将 SUT 所依赖的内容都隔离后，仅仅测试 SUT。

群居测试是指，将 SUT 和依赖模块都一起进行测试。

## 独居测试的优缺点

优点：

- 可以更精准地定位问题

  比如，在架构分层比较明确的时候，可以使用独居测试来测试每一层的代码

缺点：

- 需要写大量的 mock 代码
- 暴露了实现细节，对重构不友好
- 忽略了组件间的真实交互

## 群居测试的优缺点

优点：

- 提供更真实的环境
- 可以发现组件间的交互问题

缺点：

- 定位问题不够精准
- 准备环境和数据比较费力

> 其实这是两种不同的测试风格，选择团队更喜欢的，并保持一致性即可。

> 在架构分层不够清晰的时候，更适合采用群居测试，比如前端。
